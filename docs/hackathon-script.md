# Swan Home AI Hackathon — 活动执行脚本

## 概览

- **时间**：9:00 - 12:00（3小时）
- **参与者**：天鹅到家高管团队（CEO、部门长等）
- **分组**：4组，每组配 1 名引导员
- **目标**：每组产出一个可演示的 AI 产品原型，上台展示
- **方式**：全程 AI 驱动开发，参与者负责需求和决策，AI 负责实现

---

## 时间线

### 第一阶段：开场 & 需求收集（9:00 - 9:30）30min

#### 9:00 - 9:10 开场介绍（10min）
- 今天要做什么：每组用 AI 做出一个产品原型，最后上台展示
- 展示 SDK 的能力演示（提前录好或现场跑一个）：
  - 输入岗位信息 → 自动生成小红书文案 + 海报
  - 导入一堆阿姨数据 → 自然语言搜索匹配
- 让大家建立信心：AI 真的能做到这些

#### 9:10 - 9:30 分组 & 需求讨论（20min）
每组围绕以下问题讨论，引导员记录：

1. **你们组想做哪个方向？**（可以组合，也可以聚焦一个）
   - A. 招募获客工具 — 帮公司高效招募阿姨
   - B. 阿姨档案管理 — 把阿姨信息结构化，方便查找
   - C. 智能匹配推荐 — 客户描述需求，AI 推荐最合适的阿姨
   - D. 以上组合，或者自由发挥

2. **产品给谁用？**
   - 内部运营人员？
   - 阿姨本人？
   - C 端客户？
   - 管理层看板？

3. **风格基调选择**（每组选不同的，避免撞车）：

| 风格 | 关键词 | 适合方向 |
|------|--------|---------|
| 🌸 温馨亲和 | 粉色系、圆角、插画风、亲切感 | 面向阿姨/客户 |
| 🏢 专业商务 | 深色系、数据卡片、仪表盘风 | 管理后台/看板 |
| 🎯 简约高效 | 白底、大留白、极简交互 | 工具型产品 |
| 🎨 潮流社交 | 渐变色、卡片流、社交媒体感 | 招募/营销场景 |

> 引导员注意：确保 4 组选择不同风格，最终展示有差异化

---

### 第二阶段：AI 开发（9:30 - 11:15）105min

#### 引导员操作流程

每组一台电脑，打开 AI 编程助手（Claude Code / Cursor），加载项目仓库。

**Step 1：搭界面骨架（9:30 - 10:00）30min**

引导员带参与者对 AI 说：

> "基于这个项目的 SDK，帮我搭建一个 [风格基调] 风格的 Web 界面。
> 产品方向是 [选定方向]，用户是 [目标用户]。
> 技术栈用 React + Tailwind，先搭出页面框架和导航。"

让参与者参与决策：
- 页面要几个？每个页面放什么？
- Logo 和产品名叫什么？（让他们起名字，参与感）
- 配色喜欢哪个方向？

**Step 2：接入核心功能（10:00 - 10:45）45min**

根据选定方向，依次接入 API：

**方向 A — 招募获客：**
```
"在招募页面接入 recruit API，做一个表单：
 用户选岗位类型、填地区、薪资、联系方式、选平台，
 点击生成后展示文案和海报图片。
 要有 loading 状态，显示实时进度。"
```

**方向 B — 阿姨档案：**
```
"做一个数据导入页面，可以粘贴或上传阿姨的原始信息，
 点击处理后调 ingest API，展示结构化结果。
 再做一个搜索页面，调 search API，展示搜索结果列表。"
```

**方向 C — 智能匹配：**
```
"做一个对话式界面，用户输入需求描述，
 调 match API，展示推荐结果卡片，
 每张卡片显示阿姨姓名、评分、推荐理由。"
```

**方向 D — 组合/自由：**
引导员灵活组合以上 prompt。

> 引导员注意：每完成一个功能先跑通看效果，让参与者看到实时成果，保持兴奋感

**Step 3：打磨体验 & 自由发挥（10:45 - 11:15）30min**

核心功能跑通后，让参与者自由提需求：
- "能不能加个历史记录？"
- "这个配色换一下"
- "能不能加个对比功能？"
- "首页加个数据统计看板"

引导员鼓励大家大胆提，AI 现场实现，体验 AI 的能力边界。

---

### 第三阶段：展示准备（11:15 - 11:30）15min

每组准备 5 分钟的展示：
1. **产品名称和定位**（一句话说清楚）
2. **现场演示**（跑通核心功能，live demo）
3. **亮点**（最满意的地方是什么）
4. **感受**（用 AI 做产品的体验）

引导员帮每组理一下展示逻辑，确保 demo 能跑通。

---

### 第四阶段：上台展示 & 总结（11:30 - 12:00）30min

- 每组 5 分钟展示 + 2 分钟提问（4 组共 28 分钟）
- 最后 2 分钟总结

---

## 引导员 Checklist

### 活动前准备
- [ ] 每组一台电脑，已安装 Node.js + Bun
- [ ] 项目仓库已 clone，`bun install` 已完成
- [ ] `.env` 已配置好 API keys
- [ ] `bun test bdd/` 跑通，确认 API 正常
- [ ] AI 编程工具已就绪（Claude Code / Cursor）
- [ ] 提前跑一遍完整流程，确保网络通畅

### 现场注意
- 参与者是高管不是工程师，不要讲技术细节
- 重点让他们做决策（选方向、选风格、提需求），而不是看代码
- 每完成一个功能就展示效果，保持节奏感和成就感
- 如果某个 API 超时或失败，不要慌，重试一次或跳过先做下一个
- 预留 buffer 时间，宁可提前完成也不要赶

### 应急预案
- API 全挂：提前准备好截图/录屏，展示预期效果
- 网络慢：切手机热点
- AI 生成的界面有问题：引导员手动微调，不让参与者看到报错
